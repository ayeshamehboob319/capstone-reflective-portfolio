# 2. Ethics Reflection

## Ethical Issue: Data Privacy and User Consent

In today’s digital world, data privacy has become one of the most important ethical concerns in software development. Almost every application we use collects some form of personal data, such as location, browsing history, contacts, or user preferences.

### What is the Problem?

The main problem is the lack of transparency and meaningful user consent. Most users simply click “agree” on long and complex privacy policies without actually understanding them. As a result, personal data can be misused, shared without clear permission, or even sold to third parties for profit.

This not only violates user trust but can also lead to serious consequences like:

* Identity theft
* Data breaches
* Loss of privacy

### Which Professional Code Applies?

This issue is clearly addressed in both the ACM and IEEE Codes of Ethics:

* IEEE emphasizes public safety, honesty, and protecting user privacy.
* ACM focuses on fairness, responsibility, and respecting user confidentiality.

These codes highlight that computing professionals must:

* Protect user privacy and confidential data
* Be transparent and honest about how data is used
* Avoid actions that may harm users
* Ensure users give informed and meaningful consent

### My Decision and Justification

If I were designing a system, I would make sure that users clearly understand what data is being collected and why. Instead of long and confusing policies, I would present privacy information in a simple and easy-to-understand way.

I would also:

* Allow users to accept, reject, or customize data sharing
* Minimize unnecessary data collection
* Avoid sharing data with third parties unless absolutely required

For me, building ethical systems means putting user trust and rights above profit.
